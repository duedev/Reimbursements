Advanced tuning
Sweet spots. OpenRouter's free models allow about 20 requests/min and 50 requests/day; bursting past either returns a 429 the pipeline can only show as failed receipts. Keep the per-minute cap a little under 20 and let “wait out a 429” ride out the short bucket refill. On a local / Docker server there's no quota — turn both off so nothing throttles.

Adding $10 of credit to OpenRouter raises the free-model daily cap from 50 to 1000 requests/day — those 1000 are still free (credit only pays for non-free models). The per-minute cap stays ~20.

Enabling Multi-User Mode
  1. Set MULTIUSER_ENABLED=true in your environment (in your .env file or docker-compose.yml under environment:).
  2. Restart the server — docker compose restart app or stop/start the Python process.
  3. On first launch, navigate to /setup in your browser to create the admin account.
  4. Create additional users from Settings (admin-only section, visible when signed in as admin).
  5. Each user gets a fully isolated workspace — receipts, results, and run logs are kept separate.

Note: The AI model, server endpoint, and rate-limit settings are shared across all users — configure them in Settings → AI Model (admin view).

Google Drive Receipt Import
  1. Go to Settings → Google Drive Intake and click Connect Google Drive. Complete the OAuth flow in the browser window that opens. Alternatively, paste a refresh token directly.
  2. Set the Folder ID to the Google Drive folder you want to use as your receipts inbox. The folder ID is the last part of the folder's URL in Drive.
  3. Set a poll interval (e.g. 5 minutes). The app will check the folder on that schedule and import any new image or PDF files.
  4. Drop receipt photos into that Drive folder from your phone, computer, or the Drive app — they appear automatically on the Workspace board.
  5. Optional — Gmail→Drive bridge: use the Gmail filter (see the Gmail Receipt Filter guide) to copy receipt emails' attachments into the same folder automatically.

Google Drive is an opt-in cloud capture source. The receipt files move from Drive into the local pipeline — no data is sent anywhere else. Disconnect any time from Settings → Google Drive Intake → Disconnect.

Gmail Receipt Filter
  1. Go to Settings → Email Intake and click Download Gmail filter to download the filter XML file.
  2. In Gmail: open Settings → See all settings → Filters and Blocked Addresses → Import filters, then upload the XML file.
  3. The filter labels matching receipt emails with the label "Receipts" and skips the inbox, keeping your inbox clean.
  4. Configure the Email Intake settings to poll the Receipts label (set Mailbox / Label to Receipts).
  5. All receipt emails labelled by the filter — fuel, store confirmations, etc. — are then auto-imported each poll cycle.

The filter matches by keywords (e.g. "receipt", "purchase confirmation", "your order") rather than sender domain — so it captures receipts from any store, including those that use privacy-relay or forwarding addresses.

Email Receipt Import (IMAP)
  1. Works best with Gmail + App Password (no OAuth required). In your Google Account: Security → 2-Step Verification → App Passwords → create one for "Mail".
  2. In Settings → Email Intake: enter your Gmail address, paste the App Password, and set the Mailbox / Label to Receipts (or INBOX to read all mail).
  3. Set a poll interval (e.g. 5 minutes) and click Save settings, then Test connection to verify.
  4. Forward any receipt emails to your Gmail address. The app polls the mailbox and imports each email's attachments (images / PDFs) and body text as receipts.
  5. Combine with the Gmail Receipt Filter (see above) to automatically label incoming receipt emails so only relevant mail is imported.

Email is processed locally — the app connects directly to Gmail via IMAP (no cloud intermediary). Your Gmail App Password is stored in the local secrets file, never in the main config.

About

This app reads your receipt photos and emails, extracts vendor, date, and amount using built-in OCR and an AI model (running on-host or via OpenRouter's free cloud models), and generates a polished Excel reimbursement report. Processing happens locally by default; AI calls go to your configured model endpoint — on-host, Docker bundled, or OpenRouter cloud (opt-in).

Privacy: Receipt images may be sent to your configured model endpoint. When using OpenRouter, images are sent to OpenRouter's servers unless you set the "send OCR text only" mode in Settings → AI Model. The on-host and Docker bundled modes keep all data on your machine.

Getting Started
  1. Upload receipts using the "Add Receipts" card or drag and drop
  2. Wait for AI extraction to complete (watch the board update live)
  3. Review results — click any card to correct vendor, date, amount, or category
  4. Click "Generate Report" to export a polished Excel workbook
Importing from a scan app

Scan apps like CamScanner, Adobe Scan and Genius Scan already crop, de-skew and enhance each page, so this app skips its own auto-crop and just reads them. Each PDF page (or photo) becomes one receipt — you can drag exported files straight onto the Add Receipts card, or use the guided import below to apply the best scanned-document settings automatically.

  • Export as PDF (multi-page = one receipt per page) or JPG at HD / original quality
  • Color mode “Original” or “B&W document” — both read well
  • A free-tier watermark is fine — it sits in the footer and doesn't affect the total
Docker Commands
Using the Docker bundled LLM

A fully-local, private, OpenAI-compatible LLM (llama.cpp serving a vision GGUF) ships with this app as the model-server service in docker-compose, under the bundled-llm profile, on port 11434. The model weights are baked into the image (~2–3 GB), so it works completely offline — nothing leaves your machine.

1. Start it

When 🐳 Docker bundled LLM is selected in Settings → AI Model you can also use the Start / Stop / Restart / Load buttons there instead of the command line.

2. Wire it up
  1. Settings → AI Model → pick 🐳 Docker bundled LLM.
  2. The Server URL auto-fills — http://model-server:11434/v1 inside Docker, or http://127.0.0.1:11434/v1 on the host.
  3. Click ↻ Refresh on the model list, then select the model.
3. Swap the model

Rebuild the image with build args MODEL_URL / MMPROJ_URL (default alias qwen3-vl-2b-instruct) to bake in a different vision GGUF.

Troubleshooting

If it shows as unreachable: make sure the bundled-llm profile is up (docker compose ps), click 🔎 Auto-detect in the AI Model card, and check the model-server logs.

Pipeline Overview

Every receipt runs through the same ordered steps. Each is tagged with how it works: rules-based = no AI, runs offline; AI / model = calls your configured model endpoint; image stays local / sends the image shows whether the receipt picture is sent to the endpoint. Steps 3 (LLM OCR) and 5 (vision rescue) send the image to your model endpoint — local, Docker, or OpenRouter depending on your Settings.

  1. Upload & queue rules-basedimage stays local Files are staged to a working folder and obvious duplicates are detected before processing.
  2. Auto-rotate (EXIF) rules-basedimage stays local Bakes the camera's orientation tag into the actual pixels so the receipt is upright.
  3. Grayscale rules-basedimage stays local Converts to high-contrast black & white for cleaner, more reliable reading.
  4. Auto-crop rules-basedimage stays local Trims the uniform photo border around the receipt (skipped for scan-app imports, which are already cropped).
  5. OCR-guided rotation check rules-basedimage stays local Tries the three 90° rotations and keeps whichever the OCR engine reads best — no model involved.
  6. Built-in OCR (RapidOCR) rules-basedimage stays local The primary text reader, running fully offline — produces the text plus per-line boxes. The image never leaves your machine.
  7. LLM-OCR cross-reference (optional) AI / modelsends the image A vision model also transcribes the receipt so the extractor can cross-check both readings. Turn it on with the “Also use this model for OCR” toggle, or force it for one receipt with Retry with AI OCR. Skipped or bounded automatically under rate-limits.
  8. Distillation AI / modeltext only — no image Turns the OCR text into structured fields (vendor, date, amount, category, summary) and grounds the amount in the printed TOTAL. On a transient rate-limit (429) it waits for the bucket to refill (bounded) before giving up.
  9. Offline fallback parser rules-basedimage stays local A regex parser fills the fields when the model is unavailable or throttled, so a receipt is never lost.
  10. Vision rescue (last resort) AI / modelsends the image Only when the OCR text is missing or low-confidence: a vision model reads the receipt image directly.
  11. Field markup rules-basedimage stays local Maps the final vendor / date / amount back to the OCR line each came from — these are the colour boxes drawn on the image in review.
  12. Classify, audit & score rules-basedimage stays local Assigns the category, audits the amount against the OCR text, scores confidence, and de-duplicates.
  13. Rename rules-basedimage stays local The file is renamed category_date_vendor for a tidy archive.
  14. Review & approve youimage stays local You verify and correct each field in the review window, then approve.
  15. Export rules-basedimage stays local Builds a multi-sheet Excel workbook with charts and per-category receipt images. Image compression happens here — never before OCR, so reading always uses full-resolution images.
Tips
  • Good lighting and a flat surface give better OCR results
  • You can edit any field in the review window before approving
  • The report includes charts and a category breakdown
  • Pick where the model runs in Settings → AI Model: OpenRouter (free cloud) or an on-host LLM
